# 游客访问工作台题目功能调整

## 改动背景

当前工作台页面在未登录状态下会同时请求题目数据和个人提交数据。由于提交相关接口需要登录，游客进入页面时会触发不必要的鉴权流程，导致页面行为不符合“游客可看题、不可提交”的预期。

本次调整的目标是：

1. 未登录用户仍然可以在工作台查看题目列表和题目详情。
2. 未登录用户不能发起样例测试和正式提交。
3. 未登录状态下不再加载、展示或复用任何个人提交缓存。

## 改动文件清单

1. `programming-frontend/src/hooks/use-submissions.ts`
2. `programming-frontend/src/components/ProblemDetail.tsx`
3. `programming-frontend/src/lib/i18n/zh.ts`
4. `programming-frontend/src/lib/i18n/en.ts`

## 具体改动内容

### 1. 调整提交记录 Hook 的游客逻辑

文件：`programming-frontend/src/hooks/use-submissions.ts`

- 新增对登录状态的读取。
- 在未登录时直接跳过提交记录请求，只返回空数组。
- 在登录态丢失时主动清空本地缓存中的提交记录，避免游客看到旧的个人数据。
- 在前端提交代码前增加登录判断，未登录时直接提示，不再请求后端评测接口。

### 2. 调整题目详情页的游客交互

文件：`programming-frontend/src/components/ProblemDetail.tsx`

- 新增游客提示条，明确说明“可以浏览题目，但测试和提交需要登录”。
- 根据当前路由自动跳转到桌面端或移动端登录页。
- 对“样例测试”和“提交”按钮增加未登录禁用逻辑。
- 用户在未登录状态下尝试提交时，会先收到提示，再跳转登录页。

### 3. 补充中英文文案

文件：

- `programming-frontend/src/lib/i18n/zh.ts`
- `programming-frontend/src/lib/i18n/en.ts`

新增游客模式相关提示文案，保证桌面端和移动端在中英文环境下都能正确展示。

## 改动原因说明

这次改动优先选择前端处理，原因如下：

1. 当前需求本质上是工作台页面的游客展示策略调整。
2. 后端的题目读取接口本身已经支持公开访问，不需要额外修改接口权限。
3. 在前端拦截提交和测试行为，可以快速满足“只看题、不提交”的产品诉求。
4. 同时清理游客态下的提交缓存，可以避免历史用户数据误展示的问题。

## 预期效果

完成后，游客进入工作台时应表现为：

- 可以正常看到题目列表。
- 可以点进题目查看描述、输入输出格式、约束和示例。
- 代码区仍可展示，但测试和提交操作会被禁用并提示登录。
- 不会加载游客无权访问的个人提交数据。
